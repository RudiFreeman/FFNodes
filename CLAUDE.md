**Продукт:** FFmpeg Visual — open-source десктоп-приложение, визуальная обёртка над FFmpeg.
Полное продуктовое описание — **[docs/PRD.md](docs/PRD.md)**.

## 🏗 Стек (РЕШЕНО — обоснование в [docs/PRD.md](docs/PRD.md) §5)

Критерии выбора: красивый UI, стабильность, кроссплатформенность, скорость.

- **Tauri** — оболочка десктопа (кроссплатформа Mac/Win/Linux, лёгкая, быстрая, безопасный
  запуск внешних процессов). Внутри Rust, но писать Rust почти не придётся.
- **React + TypeScript** — фронтенд (UI + логика).
- **React Flow** (xyflow) — нодовый холст. Закрывает самую сложную часть «из коробки».
- **Tailwind CSS** — стилизация.
- **FFmpeg** — системный или бандленный бинарник; Tauri запускает как процесс.
- **Vite** — сборка фронтенда.

**Python НЕ используется** — отклонён осознанно (проигрывает по стабильности, упаковке под
3 ОС, скорости; готовые Python-FFmpeg либы не нужны — команду генерируем сами). См. PRD §5.

## 📅 Статус и этапы

- **Текущий этап: РАБОЧИЙ РЕНДЕР — полный цикл «файл → пайплайн → результат».**
  Каркас Tauri+React+TS + React Flow + Tailwind. Каталог как ДАННЫЕ, граф → команда.
  Выбор входа (`pick_input_file`) + метаданные (`probe_media`/ffprobe) в панели превью.
  **Рендер:** кнопка «Рендер» запускает `run_ffmpeg` (Rust) — save-диалог → запуск FFmpeg →
  потоковый прогресс (события `render-progress`, парс `out_time_us`) → линия прогресса под
  TopBar. Кнопка disabled без файла/при ошибке графа.
- **Превью характеристик «До → После»:** чистая `predict.ts` (`predictOutput(graph, info)`)
  считает итоговые характеристики из графа, пересчёт на лету; обход цепочки — общий `chain.ts`;
  `FilterDef.applyToInfo` описывает вклад операции. Ноды «Вход»/«Выход» показывают сводку
  (разрешение/fps/кодек/размер), боковая панель — полный набор «До/После» по группам. Размер
  до рендера — оценка («≈»), после рендера — точный (ffprobe). probe_media отдаёт расширенный
  набор (битрейт, профиль, аудио-частота/каналы).
- **Превью-кадр «До → После»:** панель показывает реальный кадр (раньше — заглушка-иконка).
  Rust-команда `extract_frame` (FFmpeg `-frames:v 1` → JPG в temp, чистая `build_frame_args`),
  кадр в webview через asset-протокол (узкий scope `ffmpeg-visual-frame-*.jpg`) + `convertFileSrc`.
  Чистая `frame.ts` (`videoFilterChain`) собирает vf из графа; хук `usePreviewFrame` —
  «До» при выборе файла, «После» с дебаунсом при правке графа, токены против гонок.
  Переключатель «До/После» под кадром. Тесты: фронт 83 ✅, Rust 13 ✅.
  Накопление кадров в temp решено (N-011): `cleanup_old_frames` (чистая `frames_to_cleanup`)
  оставляет окно последних N — в конце `extract_frame` (keep=4) и при старте (`setup`, keep=0).
- **Валидация несочетаемых операций (N-007):** чистая `validate.ts` (`validateGraph`) по
  декларативным тегам `FilterDef.streams` (dropsVideo/dropsAudio/needsVideo) ловит бессмыслицу
  (-vn + видеофильтр; -vn + -an). Блокирует рендер (`generate.ts` → error + invalidNodeIds);
  конфликтные ноды краснеют (рамка + значок ⚠ + tooltip), сообщение в CommandBar.
- **Запуск:** `npm install`, затем `npm run tauri dev` (десктоп) или `npm run dev` (фронт).
  Тесты — `npm test` (фронт) и `cargo test` в `src-tauri/` (Rust). Билд — `npm run build`.
- **Отмена рендера:** во время рендера кнопка «Рендер» становится красной «Отмена»
  (`cancel_render` → `kill` процесса). Общий `RenderState` в Rust (`Mutex<Option<Child>>` +
  `AtomicBool` флаг отмены) через `.manage()`; `run_ffmpeg` при отмене удаляет недописанный
  выходной файл и не шлёт `render-done`. На фронте — статус `cancelled` в `useRender`,
  `cancelledRef` отличает отмену от сбоя.
- **Редактирование графа:** удаление фильтр-ноды не рвёт цепочку — чистые
  `bridgesOnDelete`+`applyBridges` (`features/add-node/relink.ts`) мостят предшественника
  удаляемой ноды с преемником, подключены через `onNodesDelete`. Тонкость порядка: React Flow
  `deleteElements` сначала режет рёбра, потом зовёт onNodesDelete — мосты считаем по полному
  снимку `edges`. Кнопка × на фильтр-ноде (видна при наведении, `deleteElements` → тот же путь);
  вход/выход не удаляются (`deletable: false`). N-003 закрыт.
- **«Изменить размер» — пресеты (N-006):** 1080p/720p/480p/Половина/Свои размеры по короткой
  стороне (как привычное «720p»: высота для гориз, ширина для верт) — вертикаль не раздувается.
  Внутри `if()`-выражения ffmpeg `-2` не работает (буквально −2px), вторую сторону считаем
  `round(x/2)*2`. Модель: `FilterParam.showIf` (условная видимость полей), FilterNode рендерит
  enum как `<select>`.
- **Защита пути от инъекции (N-004):** чистая `safe_path`(Rust)/`safePath`(TS) — путь с
  ведущим `-` префиксуется `./`, чтобы ffmpeg/ffprobe не приняли его за флаг. Rust — в
  `probe_media`/`extract_frame`; фронт — к путям из диалогов (для args рендера).
- **Открытый техдолг (CODE_NOTES):** N-001 — стиль контролов React Flow;
  N-010 — размер файла не предсказывается точно (показываем «≈»);
  N-012 — кадр «После» для временны́х фильтров (trim/speed) может быть нерепрезентативен.
- **Следующий шаг:** видеоплеер с проигрыванием до/после (нативный `<video>` не играет
  HEVC/Main 10 — нужен транскод превью или иной подход), ещё фильтры в каталоге.
- **История завершённого — [CHANGELOG.md](CHANGELOG.md).**
- **Ближайшие шаги — [docs/PRD.md](docs/PRD.md) §8.**

## ⚠️ Важные правила

0. **⛔ Порядок разработки — [docs/WORKFLOW.md](docs/WORKFLOW.md).** Любая задача идёт по нему;
   Claude напоминает и требует соблюдения. Отступления — только по явному «идём в обход».

2. **Кроссплатформенность всегда в уме** — код и пути не должны ломаться на Mac/Win/Linux
   (разделители путей, поиск бинарника FFmpeg, права на запуск процессов).
3. **FFmpeg вызывается через Tauri** (безопасный запуск процесса), не через произвольный shell;
   аргументы команды передаём массивом, не склеенной строкой (защита от инъекций).
4. Весь код пушится в **отдельную ветку**, не в main. Автор мержит через Pull Request.
5. После каждого завершённого этапа обновлять [CHANGELOG.md](CHANGELOG.md) и раздел
   «Статус и этапы» этого файла.
6. **Журнал наблюдений** — заметил по ходу техдолг/странность не по теме задачи? Не чини на
   месте — добавь запись в `docs/CODE_NOTES.md` (создать при первой записи).
7. **Тесты — рядом с кодом, не в `tests/`** (конвенция стека). Фронт: `*.test.ts` возле
   тестируемого файла (Vitest ищет по всему `src/`). Rust: юнит-тесты в том же файле в
   блоке `#[cfg(test)] mod tests` (нужен доступ к приватным функциям). Папка `tests/`
   хранит **только реестр** `tests/README.md` (что покрыто / что нужно покрыть) — НЕ сами
   тесты. Заметил недостающий тест — допиши строку в реестр, не анализируя проект заново.
8. **README по папкам** — навигация трёхуровневая: этот файл (правила + вход) →
   [docs/README.md](docs/README.md) (карта документации) → `README.md` внутри папки-хранилища.
   Заводишь папку со своей логикой (как `tests/`, `assets/`) — положи в неё `README.md`
   и добавь строку в карту docs/README.md.
9. Все комментарии в коде — **на русском**.

## 📐 Правила модульности

Дух: **малые файлы, одна ответственность, явные зависимости.** Адаптировано под React/TS.

- **НИКОГДА** не писать весь код в одном файле. Разбивай на логические модули.
- **Feature-Sliced Design** для фронтенда:
  - `src/app/` — входная точка, провайдеры, роутинг, тема.
  - `src/widgets/` — крупные самостоятельные блоки UI (холст, панель команды, каталог).
  - `src/features/` — пользовательские сценарии (добавить инпут, применить фильтр, рендер).
  - `src/shared/` — переиспользуемое: `ui/` (атомы), `lib/` (утилиты), `api/` (вызовы Tauri), `types/`.
- **Каждый файл:** ≤ 300 строк цель / 500 строк жёсткий лимит; одна ответственность;
  комментарий-шапка с описанием назначения.
- **Компоненты:** один компонент = один файл; логику выносить в хуки (`useXxx`).
- **Генерация FFmpeg-команды** — отдельный изолированный модуль (`shared/lib/ffmpeg/`),
  чистые функции «граф → массив аргументов», легко тестируемые.

**Запрещено:**
- циклические импорты;
- `import * from` (кроме типов, где оправдано);
- глобальные мутабельные переменные (кроме констант);
- бизнес-логика внутри JSX-разметки.

Все импорты явные.
