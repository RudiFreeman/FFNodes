## 🧠 Контекст проекта

**Продукт:** FFmpeg Visual — open-source десктоп-приложение, визуальная обёртка над FFmpeg.
Полное продуктовое описание — **[docs/PRD.md](docs/PRD.md)**.

**Суть:** закинул медиа → собрал пайплайн из нод/функций с человеческими описаниями →
видишь сгенерированную FFmpeg-команду внизу (редактируема) → рендер + превью. Принцип
**прогрессивного раскрытия**: новичок работает кнопками/нодами, профи правит команду.

**Разработчик:** Алексей — не программист, работает через Claude Code. Опыта с веб-фронтендом
(React/TypeScript) почти нет — Claude объясняет по ходу и учит на реальном проекте.
**Язык общения:** Русский. **Тон:** Неформальный, «ты».

## 🏗 Стек (РЕШЕНО — обоснование в [docs/PRD.md](docs/PRD.md) §5)

Критерии выбора: красивый UI, стабильность, кроссплатформенность, скорость.

- **Tauri** — оболочка десктопа (кроссплатформа Mac/Win/Linux, лёгкая, быстрая, безопасный
  запуск внешних процессов). Внутри Rust, но писать Rust почти не придётся.
- **React + TypeScript** — фронтенд (UI + логика).
- **React Flow** (xyflow) — нодовый холст. Закрывает самую сложную часть «из коробки».
- **Tailwind CSS** — стилизация.
- **FFmpeg** — системный или бандленный бинарник; Tauri запускает как процесс.
- **Vite** — сборка фронтенда.

**Python НЕ используется** — отклонён осознанно (проигрывает по стабильности, упаковке под
3 ОС, скорости; готовые Python-FFmpeg либы не нужны — команду генерируем сами). См. PRD §5.

## 📅 Статус и этапы

- **Текущий этап: РАБОЧИЙ РЕНДЕР — полный цикл «файл → пайплайн → результат».**
  Каркас Tauri+React+TS + React Flow + Tailwind. Каталог как ДАННЫЕ, граф → команда.
  Выбор входа (`pick_input_file`) + метаданные (`probe_media`/ffprobe) в панели превью.
  **Рендер:** кнопка «Рендер» запускает `run_ffmpeg` (Rust) — save-диалог → запуск FFmpeg →
  потоковый прогресс (события `render-progress`, парс `out_time_us`) → линия прогресса под
  TopBar. Кнопка disabled без файла/при ошибке графа.
- **Превью характеристик «До → После»:** чистая `predict.ts` (`predictOutput(graph, info)`)
  считает итоговые характеристики из графа, пересчёт на лету; обход цепочки — общий `chain.ts`;
  `FilterDef.applyToInfo` описывает вклад операции. Ноды «Вход»/«Выход» показывают сводку
  (разрешение/fps/кодек/размер), боковая панель — полный набор «До/После» по группам. Размер
  до рендера — оценка («≈»), после рендера — точный (ffprobe). probe_media отдаёт расширенный
  набор (битрейт, профиль, аудио-частота/каналы).
- **Превью-кадр «До → После»:** панель показывает реальный кадр (раньше — заглушка-иконка).
  Rust-команда `extract_frame` (FFmpeg `-frames:v 1` → JPG в temp, чистая `build_frame_args`),
  кадр в webview через asset-протокол (узкий scope `ffmpeg-visual-frame-*.jpg`) + `convertFileSrc`.
  Чистая `frame.ts` (`videoFilterChain`) собирает vf из графа; хук `usePreviewFrame` —
  «До» при выборе файла, «После» с дебаунсом при правке графа, токены против гонок.
  Переключатель «До/После» под кадром. Тесты: фронт 92 ✅, Rust 13 ✅.
  Накопление кадров в temp решено (N-011): `cleanup_old_frames` (чистая `frames_to_cleanup`)
  оставляет окно последних N — в конце `extract_frame` (keep=4) и при старте (`setup`, keep=0).
- **Каталог: 29 операций.** Категории: Размер/FPS (вкл. «Отступы/леттербокс» pad), Обрезка,
  Поворот/Отражение (вкл. «Повернуть на угол» rotate=°*PI/180), Скорость,
  Цвет/Яркость (вкл. «Резкость» unsharp), Конвертация/Сжатие (вкл. «Сменить кодек» H.264/H.265/VP9),
  Экспорт GIF, Звук (`audio.ts` — «Громкость», «Затухание звука» afade, «Монофонический звук» -ac 1,
  «Нормализация звука» loudnorm),
  Эффекты (`effects.ts` — «Затухание», «Размытие» gblur, «Виньетка» vignette, «Реверс»),
  Слияние (`merge.ts` — «Наложить поверх» overlay, «Склеить ролики» concat).
  Модель `CommandContribution`: `vf` (видеофильтр) + `af` (аудиофильтр) + `outputArgs`;
  генератор собирает `-vf`/`-af`/опции. Добавить операцию = файл в `catalog/` + строка в `index.ts`.
- **Multi-input графы — DAG + filter_complex (N-008, N-016).** Граф больше не только линейная
  цепочка: несколько входов, ветвление, слияние потоков. Рубильник `dag.isLinearGraph(graph)`
  отделяет «простой -vf путь» (линейный граф — генерация как раньше, 27 single-фильтров не
  тронуты) от filter_complex-пути (merge-операция / >1 входа / ветвление). `dag.ts` — topoSort
  (алгоритм Кана) + isLinearGraph; `complex/` — построитель filter_complex (`buildComplexPlan`)
  с аллокатором лейблов потоков (`v1`/`a1`/`0:v`). Merge-операция объявляет необязательное поле
  `FilterDef.merge` (MergeSpec: `videoInputs`, `toComplex` строит фрагмент с лейблами,
  `applyMerge` для предсказания, `inputLabels` для UI). **GIF** — single-input merge (палитра
  split→palettegen→paletteuse, N-008 закрыт). **overlay/concat** — два входа. Merge-нода
  (`MergeNode.tsx`) рисует ДВА именованных target-handle; порядок входов разрешается по
  `GraphEdge.targetHandle` (in-0 основной). Кнопка «Добавить вход» (каталог) + выбор файла на
  доп. input-ноде; путь входа хранится в `params.path`. onConnect: один источник на хэндл.
  relink при удалении merge-ноды мостит только основной вход. predict — топо-обход с накоплением
  MediaInfo по нодам. Превью-кадр «После» для DAG — Rust `extract_frame_complex` (N-016 закрыт).
- **Валидация несочетаемых операций (N-007, N-014):** чистая `validate.ts` (`validateGraph`) по
  декларативным тегам `FilterDef.streams` (dropsVideo/dropsAudio/needsVideo/needsAudio) ловит
  бессмыслицу (-vn + видеофильтр; -an + аудиофильтр; -vn + -an). Плюс правило 4 (N-014): дубль
  «однозначного» выходного флага (-c:v/-c:a/-f) у двух операций — ловится из их `outputArgs`,
  без тегов. Плюс правило DAG: merge-нода (videoInputs≥2) с незаполненным вторым входом.
  Работает по `topoSort` (поддержка ветвления/слияния). Блокирует рендер (`generate.ts` →
  error + invalidNodeIds); конфликтные ноды краснеют (рамка + значок ⚠ + tooltip).
- **Запуск:** `npm install`, затем `npm run tauri dev` (десктоп) или `npm run dev` (фронт).
  Тесты — `npm test` (фронт) и `cargo test` в `src-tauri/` (Rust). Билд — `npm run build`.
- **Отмена рендера:** во время рендера кнопка «Рендер» становится красной «Отмена»
  (`cancel_render` → `kill` процесса). Общий `RenderState` в Rust (`Mutex<Option<Child>>` +
  `AtomicBool` флаг отмены) через `.manage()`; `run_ffmpeg` при отмене удаляет недописанный
  выходной файл и не шлёт `render-done`. На фронте — статус `cancelled` в `useRender`,
  `cancelledRef` отличает отмену от сбоя.
- **Редактирование графа:** удаление фильтр-ноды не рвёт цепочку — чистые
  `bridgesOnDelete`+`applyBridges` (`features/add-node/relink.ts`) мостят предшественника
  удаляемой ноды с преемником, подключены через `onNodesDelete`. Тонкость порядка: React Flow
  `deleteElements` сначала режет рёбра, потом зовёт onNodesDelete — мосты считаем по полному
  снимку `edges`. Кнопка × на фильтр-ноде (видна при наведении, `deleteElements` → тот же путь);
  вход/выход не удаляются (`deletable: false`). N-003 закрыт.
- **«Изменить размер» — пресеты (N-006):** 1080p/720p/480p/Половина/Свои размеры по короткой
  стороне (как привычное «720p»: высота для гориз, ширина для верт) — вертикаль не раздувается.
  Внутри `if()`-выражения ffmpeg `-2` не работает (буквально −2px), вторую сторону считаем
  `round(x/2)*2`. Модель: `FilterParam.showIf` (условная видимость полей), FilterNode рендерит
  enum как `<select>`.
- **Защита пути от инъекции (N-004):** чистая `safe_path`(Rust)/`safePath`(TS) — путь с
  ведущим `-` префиксуется `./`, чтобы ffmpeg/ffprobe не приняли его за флаг. Rust — в
  `probe_media`/`extract_frame`/`extract_frame_complex` (каждый вход); фронт — к путям из
  диалогов (для args рендера).
- **Оценка размера файла (N-010 закрыт):** `size.ts` — `estimateSize` = (видео+аудио битрейт)
  ×длительность/8; операции корректируют `video_bitrate` в applyToInfo (scale/pad/fps ∝
  пикселям/fps; compress — от CRF; changeCodec — H.265×0.6/VP9×0.65; extract/remove_audio
  обнуляют дорожку). predict пересчитывает размер только при изменении битрейта/длительности
  (иначе — реальный размер входа). Остаётся ОЦЕНКОЙ (помечена «≈»): зависит от контента.
- **Спринт 1 — UX-доводки (пункты 1–2 из docs/IDEAS.md):** (1) **Drag & drop файла** — видео
  перетаскивается в окно, грузится как через диалог. Хук `useFileDrop` (Tauri v2
  `onDragDropEvent`) → `useInputFile.acceptDroppedPath` (общий `loadPath` + `safePath` N-004 +
  отсев не-видео `videoExtensions.ts`). Оверлей «Отпусти файл здесь» в PreviewPanel
  (`pointer-events-none`). Не-видео не сбрасывает уже выбранный файл. (2) **Навигация холста** —
  панорама на ПКМ (`panOnDrag={[2]}`), ЛКМ-рамка выделения (`selectionOnDrag`), пан колесом
  (`panOnScroll`), контекстное меню погашено; **хэндлы крупнее** (общий `shared/ui/handleClass.ts`,
  ~7→14px+бордер). (3) **Каталог: размеры/иерархия** — заголовок раздела главнее пунктов
  (UPPERCASE, полужирный, у края), пункты с отступом-«лесенкой» (`pl-6`) и легче (13/11px);
  раскрытый раздел — пункты на тёмной панели (`bg-bg/40`) во всю ширину (`-mx-2`). Спринт 1 закрыт.
- **Спринт 2 — дефолты в нодах из метаданных входа:** новая нода открывается с параметрами из
  info входа (`probe_media`), а не статикой. Единый механизм — необязательное
  `FilterDef.defaultsFromInfo(info)` (чистая «info → частичные параметры»), накладывается ПОВЕРХ
  статичных `default` в `useGraph.addFilterNode` (момент ДОБАВЛЕНИЯ ноды; смена файла старые
  ноды не пересчитывает — предсказуемо, не затирает правки). Дефолт из входа у 5 операций:
  «Сменить кодек» (текущий кодек: h264→H.264 и т.д.), «Кадрировать» (полный кадр входа),
  «Отступы» (размеры входа), «Сменить частоту» (fps входа округл.), «Обрезать по времени»
  (конец=длительность); остальные 24 — без дефолта осознанно (нет параметров либо творческий/
  качественный выбор). Ключ возвращается только при ненулевом поле info (fps отсекает 0 —
  иначе невалидный `fps=0`). Пункт роадмапа «панель только выбранной ноды» снят: премиса не
  совпала с кодом (параметры рисуются ВНУТРИ нод на холсте). Спринт 2 закрыт.

## 📐 Правила модульности

Дух: **малые файлы, одна ответственность, явные зависимости.** Адаптировано под React/TS.

- **НИКОГДА** не писать весь код в одном файле. Разбивай на логические модули.
- **Feature-Sliced Design** для фронтенда:
  - `src/app/` — входная точка, провайдеры, роутинг, тема.
  - `src/widgets/` — крупные самостоятельные блоки UI (холст, панель команды, каталог).
  - `src/features/` — пользовательские сценарии (добавить инпут, применить фильтр, рендер).
  - `src/shared/` — переиспользуемое: `ui/` (атомы), `lib/` (утилиты), `api/` (вызовы Tauri), `types/`.
- **Каждый файл:** ≤ 300 строк цель / 500 строк жёсткий лимит; одна ответственность;
  комментарий-шапка с описанием назначения.
- **Компоненты:** один компонент = один файл; логику выносить в хуки (`useXxx`).
- **Генерация FFmpeg-команды** — отдельный изолированный модуль (`shared/lib/ffmpeg/`),
  чистые функции «граф → массив аргументов», легко тестируемые.

**Запрещено:**
- циклические импорты;
- `import * from` (кроме типов, где оправдано);
- глобальные мутабельные переменные (кроме констант);
- бизнес-логика внутри JSX-разметки.

Все импорты явные.
