# 🎬 FFmpeg Visual

![Status](https://img.shields.io/badge/status-early--stage-orange) ![License](https://img.shields.io/badge/license-open--source-green) ![Tauri](https://img.shields.io/badge/tauri-2-blue) ![React](https://img.shields.io/badge/react-18-blue) ![TypeScript](https://img.shields.io/badge/typescript-strict-blue) ![React Flow](https://img.shields.io/badge/react--flow-xyflow-blue) ![Vite](https://img.shields.io/badge/vite-5-blue) ![Tailwind](https://img.shields.io/badge/tailwindcss-3-blue) ![FFmpeg](https://img.shields.io/badge/ffmpeg-cli-blue) ![Cross-platform](https://img.shields.io/badge/mac%20%7C%20win%20%7C%20linux-yes-blue)

Open-source визуальная обёртка над FFmpeg.
Закидываешь медиа → собираешь пайплайн из нод и функций с человеческими описаниями «что это и зачем» → видишь сгенерированную FFmpeg-команду внизу (её можно читать и править) → рендеришь и смотришь результат. Никаких длинных команд наизусть — но и ничего не спрятано от тех, кто хочет видеть всё.

---

## 🎯 Цель продукта

Дать любому человеку доступ ко всей мощи FFmpeg без необходимости запоминать длинные команды — и одновременно научить понимать, что происходит «под капотом». Инструмент, на котором наглядно изучаешь все возможности FFmpeg.

---

## 👤 Целевая аудитория

Двойная, объединённая одним интерфейсом по принципу **прогрессивного раскрытия**:

- **Новички, боящиеся CLI** — работают кнопками, нодами и пресетами; команду можно не видеть.
- **Профи, знающие FFmpeg** — устали писать длинный `-filter_complex` руками; команда видна и редактируема.

Полное продуктовое описание — [docs/PRD.md](docs/PRD.md).

---

## 🚀 Статус

**Ранняя стадия — документы и подготовка.** Изучены конкуренты, зафиксировано продуктовое видение, выбран стек. Каркас приложения ещё не создан.

Ближайшие шаги — [docs/PRD.md](docs/PRD.md) §8. Порядок разработки — [docs/WORKFLOW.md](docs/WORKFLOW.md).

> Ниша осознанная: рынок «конвертеров с пресетами» переполнен (HandBrake, Videomass, Shutter Encoder…), а ниша **визуального конструктора filtergraph** почти пуста. Туда и целимся. Сравнение конкурентов — [docs/PRD.md](docs/PRD.md) §4.

## 📦 Стек

| Слой | Технология |
|---|---|
| Оболочка десктопа | Tauri (кроссплатформа Mac/Win/Linux, лёгкая, быстрая) |

## 🗓 Роадмап (MVP)

Цель MVP — доказать поток «закинул → собрал нодами → увидел команду → отрендерил».

- [ ] Drag & drop медиафайла + превью входа
- [ ] Каталог из ~10–15 частых операций с описаниями (конвертация, сжатие, обрезка, ресайз/fps, GIF, извлечение аудио, текст/водяной знак, поворот, кроп)
- [ ] Нодовый холст: input → filter → output
- [ ] Генерация корректной FFmpeg-команды из графа (видна, копируется)
- [ ] Render → запуск локального FFmpeg → результат + превью

Что отложено на потом (двусторонняя синхронизация «команда ↔ ноды», batch, timeline, полный охват фильтров) — [docs/PRD.md](docs/PRD.md) §6.

## 🛠 Запуск локально

> ⚠️ Каркас ещё не создан — команды появятся после инициализации проекта. Ориентир:

```bash
git clone <repo-url>
cd FFmpeg_visual
npm install
npm run tauri dev     # запуск десктоп-приложения в dev-режиме
npm run test          # vitest
npm run build         # production-сборка фронтенда
npm run tauri build   # сборка установщиков под текущую ОС
```

Требуется установленный **FFmpeg** в системе (или будет бандлиться в приложение — решится на этапе сборки).
