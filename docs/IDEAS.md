## 🔥 Горячие (хочу в ближайшее время)

Идеи, которые почти готовы стать задачей.

- **Панель метаданных файла (под превью, в левой панели).** Когда добавили видео — под окном
  превью показывать характеристики файла: разрешение, кодек видео/аудио, длительность, FPS,
  битрейт, размер, контейнер. Зачем: новичку понять «что у меня за файл», а нам — подставлять
  разумные дефолты в параметры нод и валидировать (напр. не ресайзить вверх). Технически: уже
  заложенная Tauri-команда `probe_media(path)` → ffprobe (`-show_format -show_streams -of json`),
  парс JSON. Требует установленного FFmpeg/ffprobe. Делать вместе с Rust-стороной превью.

- **Несколько выходов из одного входа (мульти-аутпут).** Один вход → несколько выходных нод,
  у каждой свои настройки кодирования/формата, рендерятся параллельно. Сценарий: взял одно видео
  и выгнал сразу в нескольких форматах/разрешениях (напр. 1 вход → 5 выходов, каждый со своими
  настройками). Каждый выход — отдельно настраиваемый и сохраняемый. Базовая механика, на которой
  потом строятся проекты и пресеты. **Открытый вопрос архитектуры (решить при планировании):**
  одна FFmpeg-команда с несколькими `-map`/выходными файлами (декод один раз, эффективнее, но
  сложнее генератор) **vs** несколько параллельных запусков FFmpeg (проще код и прогресс, но
  декод дублируется). Затрагивает модель графа (сейчас выход один), генератор команды, прогресс,
  превью. Крупная подсистема.

- ✅ **Сохранение / загрузка проекта + пресеты.** (Спринт 4) Всё состояние холста (вход + все
  выходы + их настройки) сохраняется в файл `.ffvproj` (JSON, версионируемая схема) и открывается
  обратно. Чистый модуль `shared/lib/project/` (serialize/deserialize, round-trip), Rust
  write/read проекта (путь из диалога), `useProject` + кнопки Открыть/Сохранить + имя проекта в
  TopBar. Пути абсолютные; пропавший вход не роняет приложение. **Пресеты** — настройки одной
  выходной ветки (операции+params без файла): сохранить ветку / применить к выходу (панель в
  каталоге, хранятся в app-config/presets). Список последних проектов (`recent.json`) —
  выпадашка «Недавние» (задел под приветственное окно). 🔒 safePath к путям из недоверенного
  файла, санация имён пресетов от path traversal.

- **Приветственное окно (как в Cursor).** При запуске — экран с вариантами: «Новый проект» /
  «Открыть проект» / внизу список последних проектов. Точка входа в приложение вместо сразу
  пустого холста. Зависит от подсистемы проектов (см. выше).

- ✅ **Дефолтные значения в нодах из метаданных входа.** (Спринт 2) Новая нода открывается с
  параметрами, заполненными из info входа (`probe_media`), а не статикой. Единый механизм —
  необязательное `FilterDef.defaultsFromInfo(info)`, накладывается поверх статичных `default`
  в `useGraph.addFilterNode` (момент добавления ноды). Дефолт из входа у 5 операций: «Сменить
  кодек» (текущий кодек), «Кадрировать», «Отступы», «Сменить частоту», «Обрезать по времени»;
  остальные 24 — без дефолта осознанно. Везде выбор кликом, как и задумано.

## 💡 Интересно (когда-нибудь)

- **Двусторонняя синхронизация «команда ↔ ноды».** Профи правит FFmpeg-команду внизу руками →
  граф нод перестраивается под неё. Самая сложная часть (парсинг произвольной команды в граф),
  поэтому вынесена за MVP. Большая ценность для продвинутых пользователей.
- **Библиотека готовых графов (рецептов).** Набор примеров «как сделать GIF», «сжать для Telegram»,
  «наложить субтитры» — открыл, подставил свой файл, отрендерил. Учит на живых примерах.
- **Шаринг/экспорт графа.** Сохранить собранный пайплайн в файл и поделиться им — чтобы переслать
  другу готовый рецепт обработки.
- **Подсказки и пояснения по фильтрам прямо в каталоге.** У каждой функции — человеческое описание
  «что это и зачем» + наглядный мини-пример до/после. Обучающий слой (ключевая идея продукта).
- **Batch-обработка (списки файлов + группы).** Применить собранный пайплайн сразу к пачке
  файлов. Сценарии: перетащил 5 файлов → обрабатываются последовательно по одним настройкам;
  **группировка** — разбить пачку на группы и каждой группе задать свои настройки (напр. два
  видео в 4K — отдельная группа, остальные иначе); **папка как вход** — указал папку, все видео
  внутри обрабатываются. Сильно меняет модель «один вход = одна нода» (вход становится списком).
  Глубину группировки решим при планировании (простые группы / папка-вход / и то и другое).
  Статус: **в долгий ящик** — сначала мульти-аутпут, проекты, UX.
- **Предпросмотр в реальном времени.** Лёгкий превью-рендер первых секунд, чтобы видеть эффект
  фильтра до полного рендера.
- **Бандлинг FFmpeg в приложение.** Чтобы пользователю не нужно было ставить FFmpeg отдельно —
  решить на этапе сборки (размер vs удобство; учесть лицензию FFmpeg).
- **Импорт команды из буфера.** Вставил готовую `ffmpeg ...` команду откуда-то → увидел её разбор
  нодами (частный случай двусторонней синхронизации).
- **Тёмная/светлая тема.** Аккуратный современный вид, переключение.

## 🎨 UX / визуал (полировка существующего)

Не новые подсистемы, а доводка того, что уже есть. Под часть пунктов Алексей пришлёт скриншоты
с примерами того, что нравится — разбираем визуально вместе.

- ✅ **Drag & drop файла во вкладку «Выбрать файл».** (Спринт 1) Перетащить файл прямо в зону
  выбора, а не только через системный диалог. Сделано: один файл, Tauri v2 `onDragDropEvent`
  + оверлей в PreviewPanel. (Батч — перетаскивание нескольких файлов — позже, см. Batch-обработка.)

- **Панель параметров показывает только выбранную ноду.** _(Уточнение, Спринт 2: премиса
  «слева вываливаются параметры всего» не совпала с кодом — параметры рисуются ВНУТРИ каждой
  ноды на холсте, а не в левой панели. Алексей: «с нодами всё гуд, пропускаем».)_ Идея
  остаётся на будущее в иной формулировке: когда нод/файлов станет много, дать отдельную панель
  свойств, редактирующую параметры только ВЫБРАННОЙ ноды (ноды на холсте — компактнее). Особенно
  важно при списках файлов (выбрал видео в списке → видишь его параметры). Не задача Спринта 2.

- ✅ **Навигация по холсту: перемещение и зоны попадания.** (Спринт 1) Панорама перенесена на
  **правую** кнопку (`panOnDrag={[2]}`), ЛКМ — выделение/таскание нод; пан колесом/трекпадом
  (`panOnScroll`); хэндлы крупнее (~7→14px + бордер). Проверено вживую — ПКМ и ЛКМ-рамка зашли.

- ✅ **Размеры окошек в каталоге функций.** (Спринт 1) Пункты ужаты, выстроена иерархия:
  заголовок-«оглавление» главнее пунктов (UPPERCASE, полужирный, у края), вложенные пункты с
  отступом-«лесенкой» (pl-6) и легче (13/11px); раскрытый раздел — пункты на тёмной панели во
  всю ширину каталога. Подобрано по скриншотам-эталону (DaVinci Resolve).

## 🤔 Подумать / открытые вопросы

- Название проекта (рабочее: FFmpeg Visual) — финальное имя для open-source релиза.
- Модель устойчивости open-source проекта (донаты? просто хобби-проект? без монетизации).
- Где брать человеческие описания фильтров: писать вручную (качественно, но долго) или
  парсить `ffmpeg -h filter=...` (полно, но сухо) — возможно гибрид.
