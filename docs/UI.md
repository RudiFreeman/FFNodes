# UI/UX — FFmpeg Visual

> Дизайн-система и раскладка интерфейса. Спроектировано с помощью скила UI/UX Pro Max
> (тип продукта: Developer Tool / Node Editor, тёмная тема).
> Стек реализации: React + Tailwind + React Flow. Статус: черновик v0.1.

---

## 1. Стиль и принципы

**Стиль: Dark Mode (OLED) — профессиональный инструмент-редактор.**
Эталон ощущения: DaVinci Resolve / Blender node editor / VS Code. Тёмный фон бережёт
глаза при долгой работе, делает превью видео и цветные ноды контрастными, выглядит «как
серьёзный инструмент». Светлой темы в MVP нет (добавим позже — см. [IDEAS.md](IDEAS.md)).

Ключевые UX-принципы (под наш продукт):
- **Прогрессивное раскрытие.** Новичок работает каталогом и нодами. Командная строка внизу
  по умолчанию свёрнута/компактна — разворачивается тем, кому она нужна. Не пугаем новичка,
  не прячем от профи.
- **Одно главное действие на экране — `Render`.** Зелёная кнопка, единственный яркий CTA.
- **Каждая функция подписана.** У любой ноды/пункта каталога — человеческое описание
  «что это и зачем» (ключевая ценность продукта, заявлена автором).
- **Стабильность раскладки.** Панели не прыгают; превью резервирует место (нет CLS).

## 2. Цветовая палитра (Developer Tool / IDE, dark)

Семантические токены (Tailwind CSS variables). Основа — slate, акцент — зелёный «run».

| Роль | Hex | Назначение |
|---|---|---|
| `--bg` | `#0F172A` | Фон приложения (slate-900) |
| `--surface` | `#1B2336` | Панели, карточки нод, каталог |
| `--surface-2` | `#272F42` | Вложенные блоки, hover-фон |
| `--border` | `#475569` | Границы панелей и нод (slate-600) |
| `--fg` | `#F8FAFC` | Основной текст (slate-50) |
| `--fg-muted` | `#94A3B8` | Вторичный текст, подписи (slate-400) |
| `--primary` | `#1E293B` | Базовые кнопки/контролы (slate-800) |
| `--accent` | `#22C55E` | 🟢 CTA «Render», успех, активная связь (green-500) |
| `--on-accent` | `#0F172A` | Текст на акценте |
| `--destructive` | `#EF4444` | Удаление ноды, ошибки рендера (red-500) |
| `--ring` | `#22C55E` | Фокус-кольцо (клавиатурная навигация) |

**Цвета нод по типу** (чтобы граф читался с одного взгляда):
- Input-нода — синий (`#3B82F6`)
- Filter-нода — фиолетовый (`#8B5CF6`)
- Output-нода — зелёный (`#22C55E`)

Правило: **цвет не единственный носитель смысла** — у каждой ноды есть ещё иконка и подпись.

## 3. Типографика

- **Шрифт интерфейса:** Inter (300–700) — чистый, технический, отлично читается мелким кеглем.
- **Шрифт командной строки / кода:** моноширинный (JetBrains Mono или системный mono) —
  для блока FFmpeg-команды и технических полей. **Табличные цифры** для таймкодов/размеров.
- **Шкала:** 12 / 14 / 16 / 18 / 24 (база 14px — типично для плотного инструмента; команда 13px mono).
- **Высота строки:** 1.5 для текста, плотнее для подписей.
- **Вес:** заголовки 600, текст 400, метки/лейблы 500.

## 4. Структура экрана

Раскладка из видения автора — четыре зоны вокруг центрального холста:

```
┌─────────────────────────────────────────────────────────────────────┐
│  TopBar:  [☰ FFmpeg Visual]   проект ▾        [⚙ Настройки] [▶ Render]│
├──────────────┬──────────────────────────────────┬───────────────────┤
│              │                                  │                   │
│   ПРЕВЬЮ      │        НОДОВЫЙ ХОЛСТ              │   КАТАЛОГ ФУНКЦИЙ  │
│              │                                  │                   │
│  ┌────────┐  │   ┌──────┐   ┌────────┐  ┌─────┐ │  🔍 поиск...      │
│  │ video  │  │   │input │──▶│ scale  │─▶│out  │ │  ─────────────    │
│  │ preview│  │   └──────┘   └────────┘  └─────┘ │  ▸ Конвертация    │
│  └────────┘  │                                  │  ▸ Обрезка        │
│  ▶ 00:12     │   (зум/пан, мини-карта в углу)    │  ▸ Размер / FPS   │
│  до / после  │                                  │  ▸ Фильтры        │
│              │                                  │  ▸ Аудио          │
│              │                                  │  каждый пункт +   │
│              │                                  │  описание «зачем» │
├──────────────┴──────────────────────────────────┴───────────────────┤
│ ⌄ Команда   $ ffmpeg -i in.mp4 -vf "scale=1280:-2" out.mp4   [копир.]│
│  (моно, редактируема; сворачивается стрелкой ⌄ — прогрес. раскрытие) │
└─────────────────────────────────────────────────────────────────────┘
```

### Зоны

| Зона | Что делает | Поведение |
|---|---|---|
| **TopBar** | бренд, имя проекта, настройки, главный CTA `Render` | фиксирована сверху; `Render` — единственный зелёный |
| **Линия прогресса** | тонкая полоса прогресса рендера **под TopBar** | видна только во время рендера; процент из `time=` FFmpeg ÷ длительность файла; зелёная (accent) |
| **Превью (слева)** | вход и результат, плеер, переключатель до/после; **под плеером — блок метаданных файла** (разрешение, кодек, длительность, FPS, битрейт — из `probe_media`/ffprobe, см. [IDEAS.md](IDEAS.md)) | резервирует место (нет скачков); ресайзится перетягиванием границы |
| **Холст (центр)** | граф нод input→filter→output; зум, пан, мини-карта | React Flow; ноды цветные с иконкой и подписью |
| **Каталог (справа)** | секция «Избранное» (всегда открыта, отделена полосой) + сворачиваемые категории-аккордеоны (по умолчанию свёрнуты) с поиском и описаниями | клик добавляет ноду; звёздочка (как эффекты в Premiere) копирует в Избранное; избранное хранится в localStorage |
| **Командная строка (низ)** | сгенерированная FFmpeg-команда, моно, редактируема, копир. | по умолчанию компактна; стрелка ⌄ разворачивает (прогр. раскрытие) |

Панели между зонами — **resizable** (тянем границы). Минимальные ширины заданы, чтобы
каталог и превью не схлопывались в нечитаемое.

## 5. Ключевые компоненты

| Компонент | Назначение | Заметки по UX |
|---|---|---|
| `TopBar` | шапка + `Render` | один яркий CTA; во время рендера — спиннер, кнопка disabled |
| `PreviewPanel` | кадр входа/результата + метаданные | `FramePreview` показывает реальный кадр (FFmpeg) с переключателем до/после; приглашение выбрать файл, пока его нет; видеоплеер с проигрыванием — позже; drag&drop сюда |
| `NodeCanvas` | холст React Flow | зум/пан, мини-карта, снап связей; пустое состояние с подсказкой |
| `Node` (Input/Filter/Output) | элемент графа | цвет по типу + иконка (Lucide) + заголовок + параметры; ≥44px зоны |
| `FilterCatalog` | каталог справа | поиск, категории-аккордеоны, описание «зачем» у каждого пункта |
| `CatalogItem` | пункт каталога | название + 1 строка описания; tooltip с подробностями; drag на холст |
| `CommandBar` | команда внизу | моно-шрифт, подсветка, кнопка «копировать», сворачивание |
| `NodeParams` | параметры ноды | инлайн-поля с видимыми лейблами и helper-текстом (не только placeholder) |
| `EmptyState` | пустой холст/превью | дружелюбная подсказка «перетащи видео сюда» + маскот/иллюстрация (потом) |

## 6. Микро-взаимодействия и доступность

- **Анимации** 150–300ms, `transform`/`opacity` (не дёргать layout); уважать `prefers-reduced-motion`.
- **Состояния** hover/active/disabled визуально различимы; `cursor-pointer` на кликабельном.
- **Фокус-кольца** видимы (зелёный ring) — полная клавиатурная навигация по холсту и каталогу.
- **Контраст** ≥ 4.5:1 для текста (палитра это обеспечивает); иконки ≥ 3:1.
- **Иконки** — только SVG (Lucide), единый стиль, одна толщина обводки. Эмодзи как иконки не используем.
- **Z-index шкала** систематическая (10/20/30/50), без `z-[9999]`.
- **Ошибки рендера** — понятное сообщение рядом с источником + путь к исправлению (не просто «error»).

## 7. Что НЕ делаем в MVP (UI)

- Светлая тема (только dark в MVP; пара light/dark — позже).
- Timeline-редактор снизу (только командная строка).
- Кастомизация раскладки/перетаскивание панелей местами.
- Темизация/брендинг под пользователя.
