# PRD — FFNodes

> Open-source визуальная обёртка над FFmpeg. Цель — дать любому человеку доступ ко
> всей мощи FFmpeg без необходимости запоминать длинные команды, и одновременно
> научить его понимать, что происходит «под капотом».

**Статус:** черновик v0.1 · **Дата:** 2026-06-15 · **Автор:** Aleksei

---

## 1. Проблема

FFmpeg — самый мощный медиа-инструмент в мире, но у него отвратительный порог входа:

- Команды длинные и нечитаемые: `ffmpeg -i in.mp4 -vf "scale=1280:-2,fps=15" -c:v libx264 ...`
- Сотни фильтров и флагов, разбросанных по документации без наглядных примеров.
- Чтобы вспомнить, как сделать GIF / обрезать / наложить водяной знак — каждый раз гуглишь.
- Даже опытные пользователи устают вручную писать `-filter_complex`.

**Боль автора (живой инсайт):** «Я не люблю писать эти длинные команды и разбираться
со всеми возможными флагами. Хочу программу: закинул видео → есть пул функций FFmpeg
с пояснениями что это и зачем → собираю результат, а команду вижу внизу».

## 2. Решение

Десктоп-приложение, которое работает по принципу **прогрессивного раскрытия**:

1. **Закидываешь медиа** (drag & drop) → видишь превью.
2. **Выбираешь функции из каталога** — каждая с человеческим описанием «что это и зачем».
3. **Собираешь пайплайн нодами** (инпут → фильтры → выход).
4. **Видишь сгенерированную FFmpeg-команду внизу** — её можно читать, копировать и править руками.
5. **Жмёшь Render** → получаешь результат + превью.

Новичок не обязан смотреть на команду. Профи правит её напрямую. Все учатся наглядно.

## 3. Целевая аудитория

Двойная аудитория, объединённая одним UI:

| Сегмент | Что им нужно | Как закрываем |
|---|---|---|
| **Новички, боящиеся CLI** | Простота, подписи, пресеты | Каталог функций с описаниями, ноды, превью |
| **Профи, знающие FFmpeg** | Не писать руками длинный `-filter_complex` | Ноды собирают команду, командная строка редактируема |

Дополнительно: продукт как **обучающий инструмент** — наглядно изучать все возможности FFmpeg.

## 4. Ключевая дифференциация (почему не «ещё один конвертер»)

Рынок FFmpeg-GUI делится на два лагеря:

- **Лагерь «конвертеры с пресетами»** — переполнен: HandBrake, Shutter Encoder, Videomass,
  QWinFF, NMKoder и десятки других. Туда лезть бессмысленно.
- **Лагерь «визуальный конструктор filtergraph»** — почти пуст. Единственный заметный
  конкурент — [FFmpeg Explorer](https://github.com/antiboredom/ffmpeg-explorer) (Svelte + xyflow),
  но автор сам называет его незаконченным, есть проблемы со сложными командами.

**Наша позиция:** гибрид нодового конструктора + каталога функций с пояснениями +
видимой/редактируемой командной строки. Этого не делает никто.

### Конкуренты (сводка)

| Проект | Тип | Стек | Open Source | Слабое место |
|---|---|---|---|---|
| HandBrake | Конвертер | C/C++/GTK | ✅ | Только транскодинг, нет filtergraph |
| Shutter Encoder | Конвертер+ | Java | ✅ | Команда скрыта, не обучает |
| Videomass | Конвертер+timeline | Python/wxPython | ✅ | Формы, не ноды; UI устарел |
| QWinFF / NMKoder | Простой фронтенд | Qt / C# | ✅ | В осн. Windows, узкий охват |
| FFmpeg Explorer | Нодовый редактор | Svelte/xyflow | ✅ | Незакончен, web-only, без каталога с описаниями |

## 5. Платформа и стек (РЕШЕНО)

Критерии выбора (заданы автором): красивый UI, стабильность, кроссплатформенность, скорость.

| Слой | Технология | Роль |
|---|---|---|
| Оболочка десктопа | **Tauri** | Кроссплатформа Mac/Win/Linux, лёгкость (~10 МБ), скорость, безопасный запуск FFmpeg |
| UI + ноды | **React + React Flow + TypeScript** | Красивый UI, готовый нодовый холст, большая экосистема примеров |
| Стилизация | **Tailwind CSS** | Быстрый аккуратный современный вид |
| Логика (граф → команда) | **TypeScript** | Генерация FFmpeg-команды, каталог фильтров с описаниями |
| Исполнение | системный/бандленный **FFmpeg** | Tauri запускает как процесс |

**Python НЕ используется.** Рассматривался Python-sidecar (логика на Python), но отклонён:
проигрывает по 3 из 4 критериев автора — стабильность (2 процесса = больше точек отказа),
кроссплатформенность (Python тяжело упаковывать под 3 ОС), скорость (запуск рантайма + IPC).
Генерация команды — это работа со строками, одинаково проста на TS; готовые Python-FFmpeg
библиотеки не нужны, т.к. мы сами формируем команду. «Все пишут на Python» относится к
скриптам, а не к кроссплатформенным десктоп-приложениям.

## 6. MVP (минимально жизнеспособный продукт)

Цель MVP — доказать концепцию «закинул → собрал нодами → увидел команду → отрендерил».

**Входит в MVP:**
- [ ] Drag & drop медиафайла + превью входа.
- [ ] Каталог из ~10–15 самых частых операций с описаниями:
      конвертация формата, сжатие, обрезка по времени, изменение размера/fps,
      экспорт в GIF, извлечение аудио, наложение текста/водяного знака, поворот, кроп.
- [ ] Нодовый холст: input-нода → filter-ноды → output-нода.
- [ ] Генерация корректной FFmpeg-команды из графа (видна внизу, копируется).
- [ ] Кнопка Render → запуск локального FFmpeg → результат + превью слева.

**НЕ входит в MVP (потом):**
- Ручное редактирование команды с обратной синхронизацией в ноды (сложно).
- Batch-обработка пачки файлов.
- Timeline-редактор.
- Полный охват всех фильтров FFmpeg.
- Кастомные пресеты пользователя / шаринг графов.

## 7. Открытые вопросы

- Стек фронтенда: React vs Svelte (зависит от выбора нодовой библиотеки).
- Бандлить FFmpeg в приложение или требовать установленный? (лицензия, размер).
- Двусторонняя синхронизация «команда ↔ ноды» — нужна ли в v1 или это v2.
- Модель монетизации / устойчивости проекта (для open-source — донаты? нет?).
- Название проекта (рабочее: FFmpeg Visual).
